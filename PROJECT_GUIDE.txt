1. 🎯 ABOUT THE PROJECT
------------------------------------------------------------

This intelligent dashboard is an AI-powered medical diagnostic system. 

It is specifically designed to predict the likelihood of a blood infection (sepsis) based on key patient health metrics such as:
🔹 WBC Count (White Blood Cells)
🔹 Heart Rate
🔹 Temperature
🔹 SpO2 (Oxygen Saturation)

Using advanced Machine Learning models trained on clinical data, it provides real-time risk predictions and visualizes patient history trends! 📈


2. 💻 TECHNOLOGIES USED AND WHY
------------------------------------------------------------

[ 🎨 Frontend ]

🟢 React.js (via Vite)
Why? React provides a fast and modern JavaScript framework used to build responsive, component-based user interfaces. Vite ensures the development server is blazing fast.

🟢 Recharts
Why? Used for robust, React-native chart rendering. We chose Recharts because it seamlessly integrates with React states and provides beautiful interactive charts (WBC, Heart rate, Risk trends) natively.

🟢 CSS3 (Vanilla)
Why? Chosen for high flexibility in custom Light/Dark theme toggling and adding engaging micro-animations.


[ ⚙️ Backend ]

🟠 FastAPI
Why? Chosen over Flask or Django because of its high performance, native asynchronous support, and automatic generation of interactive API documentation (Swagger UI).

🟠 SQLAlchemy
Why? An Object Relational Mapper (ORM) used to safely manage the database without writing manual raw SQL queries.

🟠 SQLite
Why? A lightweight, embedded database ideal for tracking patient history efficiently without needing a heavy cloud or SQL server setup.


[ 🧠 Machine Learning & AI ]

🟣 Scikit-Learn (Random Forest)
Why? Tree-based predictive algorithms are highly readable, prevent overfitting, and perform exceptionally well on tabular clinical arrays.

🟣 XGBoost (Extreme Gradient Boosting)
Why? Used alongside RF for its high-speed performance on imbalanced medical datasets and logloss optimization.

🟣 Imbalanced-learn (SMOTE)
Why? Medical data is often skewed heavily toward "Healthy". Synthetic Minority Over-sampling Technique (SMOTE) artificially balances the dataset by generating "Infected" edge cases, preventing AI bias.



3. 🔬 TECHNICAL DOCUMENTATION & METHODOLOGY
------------------------------------------------------------

🔹 Synthetic Data Generation: 
The backend automatically generates 10,500 synthetic patient records (7,000 healthy vs 3,500 infected) with carefully defined clinical ranges to train the AI.

🔹 Preprocessing: 
A `StandardScaler` standardizes the metrics (centering them around zero) so large values like Platelets don't overwhelm small values like Hemoglobin.

🔹 Balancing (SMOTE): 
Upsamples the minority 'Infection' class so our machine learning engine doesn't blindly predict "Healthy" for everything.

🔹 Hyperparameter Tuning: 
We run a `GridSearchCV` dynamically to test hundreds of configuration parameters across 5 folds (Cross Validation). It automatically picks the most mathematically accurate configuration!

🔹 Safe Overrides: 
Hard-coded rule constraints exist (e.g., Temp < 37.5 & WBC < 10k => Healthy Override) to prevent dangerous hallucinations by the AI in obviously clear, healthy cases.



4. 🛠️ STEP-BY-STEP RUN INSTRUCTIONS
------------------------------------------------------------

To run the full-stack system locally on your machine, you must run the backend and the frontend simultaneously in two separate terminals.


🚀 STEP 4.1: STARTING THE BACKEND & AI API

1️⃣ Open up a Command Prompt (cmd) or PowerShell terminal on your computer.

2️⃣ Navigate directly into the backend folder using the `cd` command:
    👉 cd path/to/Blood-infection/backend

3️⃣ (Optional) Activate your Python environment (like Conda or the included venv).

4️⃣ Install all required backend dependencies from the included requirements file. This ensures your computer has the AI libraries:
    👉 pip install -r requirements.txt

5️⃣ Run the FastAPI server using uvicorn:
    👉 uvicorn main:app --reload --host 0.0.0.0 --port 8000

    ⚠️ Leave this terminal window open! The API will now continuously run in the background on http://localhost:8000.



🌐 STEP 4.2: STARTING THE FRONTEND DASHBOARD

1️⃣ Open a SECOND, brand-new Command Prompt or PowerShell terminal.

2️⃣ Navigate directly into the frontend folder:
    👉 cd path/to/Blood-infection/frontend

3️⃣ Install the Node.js modules dependencies (you only need to do this the very first time):
    👉 npm install

4️⃣ Start the Vite React development server:
    👉 npm run dev

5️⃣ You're done! Open your web browser and go to the local link shown in the terminal, usually:
    👉 http://localhost:5173/
