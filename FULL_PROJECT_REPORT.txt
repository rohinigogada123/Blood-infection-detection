1. 🎯 WHAT IS THIS PROJECT?
------------------------------------------------------------

This intelligent dashboard is an AI-powered medical diagnostic system. 

It is specifically designed to predict the likelihood of a blood infection (sepsis) based on key patient health metrics such as:
🔹 WBC Count (White Blood Cells)
🔹 Heart Rate
🔹 Temperature
🔹 SpO2 (Oxygen Saturation)

Using advanced Machine Learning models trained on clinical data, it provides real-time risk predictions and visualizes patient history trends! 📈


2. 💻 TECHNOLOGIES USED AND WHY
------------------------------------------------------------

[ 🎨 Frontend (The User Interface) ]
🟢 React.js (via Vite): A fast and modern JavaScript framework used to build responsive, component-based user interfaces. 
🟢 Recharts: Used for robust, React-native chart rendering. We chose Recharts because it seamlessly integrates with React.
🟢 CSS3 (Vanilla): Chosen for high flexibility in custom Light/Dark theme toggling and adding engaging micro-animations.

[ ⚙️ Backend (The Server) ]
🟠 FastAPI: Chosen over Flask or Django because of its high performance, native asynchronous support, and automatic generation of APIs.
🟠 SQLite & SQLAlchemy: Used to safely manage a lightweight, embedded database ideal for tracking patient history efficiently.

[ 🧠 Machine Learning & AI ]
🟣 Scikit-Learn (Random Forest): Tree-based predictive algorithms are highly readable, prevent overfitting, and perform exceptionally well on tabular clinical arrays.
🟣 XGBoost: Used alongside RF for its high-speed performance on imbalanced medical datasets and logloss optimization.
🟣 Imbalanced-learn (SMOTE): Synthetic Minority Over-sampling Technique artificially balances the dataset to prevent AI bias.


3. 🔬 TECHNICAL METHODOLOGY
------------------------------------------------------------

🔹 Synthetic Data Generation: 
The backend automatically generates 10,500 synthetic patient records (7,000 healthy vs 3,500 infected) with carefully defined clinical ranges to train the AI.

🔹 Preprocessing: 
A `StandardScaler` standardizes the metrics (centering them around zero) so large values like Platelets don't overwhelm small values like Hemoglobin.

🔹 Hyperparameter Tuning: 
We run a `GridSearchCV` dynamically to test hundreds of configuration parameters across 5 folds (Cross Validation). It automatically picks the most mathematically accurate configuration!

🔹 Safe Overrides: 
Hard-coded rule constraints exist (e.g., Temp < 37.5 & WBC < 10k => Healthy Override) to prevent dangerous hallucinations by the AI in obviously clear, healthy cases.


4. 🛠️ STEP-BY-STEP RUN INSTRUCTIONS
------------------------------------------------------------

To run the full-stack system locally on your localhost, you must run the backend and the frontend simultaneously in two separate terminals.

🚀 STEP 1: STARTING THE BACKEND
1️⃣ Open a Terminal.
2️⃣ Navigate directly into the backend folder:
    👉 cd path/to/Blood-infection/backend
3️⃣ Install all required backend dependencies:
    👉 pip install -r requirements.txt
4️⃣ Run the FastAPI server:
    👉 uvicorn main:app --reload --host 0.0.0.0 --port 8000

🌐 STEP 2: STARTING THE FRONTEND
1️⃣ Open a SECOND Terminal.
2️⃣ Navigate directly into the frontend folder:
    👉 cd path/to/Blood-infection/frontend
3️⃣ Install Node modules (first time only):
    👉 npm install
4️⃣ Start the Vite server:
    👉 npm run dev
5️⃣ Open your browser and go to:
    👉 http://localhost:5173/


5. ❓ PROJECT-BASED Q&A (Interview Preparation)
------------------------------------------------------------

Q1: Why do we use both Random Forest AND XGBoost?
A: XGBoost builds trees sequentially to fix previous errors, performing better on skewed data. Random Forest builds independent trees, preventing overfitting on dense data. We use both so the system can contrast results!

Q2: How does the React app (frontend) talk to Python (backend)?
A: We use an asynchronous library called `axios`. When the user clicks the "PREDICT" button, `axios` bundles the patient's form data into a JSON package and fires an HTTP POST request over to `http://localhost:8000/api/predict`.
